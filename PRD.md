# PRD: Memory Palace

**作成者**: Ryusei | **日付**: 2026-03-29 | **ステータス**: ドラフト
**アイデアID**: #656 | **ドメイン**: 認知科学/記憶術 | **難易度**: Medium

## 問題定義

従来の間隔反復学習ツール（Anki、Memrise等）はテキストベースのフラッシュカードに依存しており、人間が本来持つ空間記憶能力を活用していない。記憶宮殿（場所法）は古代から知られる強力な記憶術だが、物理空間に依存するため拡張性に限界がある。3D空間内に記憶対象を配置し、空間記憶と結びつけた長期記憶定着を促進するデジタルツールは存在しない。

## 目標と成功指標

| 目標 | 指標 | ターゲット |
|------|------|----------|
| 空間記憶による記憶定着率向上 | 復習セッション正答率 | 従来フラッシュカード比 +20% |
| ユーザーの継続利用 | 週次アクティブ率 | 登録後4週間で 60% 以上 |
| 忘却曲線の個人適応 | 予測復習タイミングの的中率 | 80% 以上（±1日以内） |
| MVP機能の完成 | コア機能実装完了 | 2週間以内 |

## スコープ外

- マルチプレイヤー / ソーシャル機能
- モバイルアプリ（ネイティブ）
- VR/AR デバイス対応
- 有料プラン / 課金機能
- 多言語対応（MVP は日本語のみ）
- 高度なML モデル（MVP はルールベース）

## ユーザーストーリー

- 学習者として、3D空間内にルームを作成し、記憶したいアイテムを空間内に配置したい。なぜなら空間的な文脈と結びつけることで長期記憶に定着させたいから。
- 学習者として、忘却曲線に基づいた最適なタイミングで復習通知を受けたい。なぜなら効率的に記憶を維持したいから。
- 学習者として、復習セッションの正答率と忘却曲線の推移を可視化したい。なぜなら自分の記憶パターンを把握し学習戦略を改善したいから。
- 学習者として、3D空間を歩き回って配置した記憶アイテムを「再訪」したい。なぜなら空間記憶を活性化して想起を促進したいから。

## 技術要件

### 技術スタック

- **フロントエンド**: TypeScript / React + Three.js（3D レンダリング）
- **バックエンド**: Python / FastAPI（API 層 + ML/データ処理）
- **データベース**: PostgreSQL
- **インフラ**: GCP Cloud Run
- **ビルドツール**: Vite（フロントエンド）、Ruff（Python リンター）

### アーキテクチャ

Python/FastAPI + React + Three.js SPA + PostgreSQL

- **フロントエンド**: React SPA + Three.js で 3D 記憶宮殿を描画。REST API でバックエンドと通信。
- **バックエンド**: FastAPI で REST API を提供。忘却曲線モデルと復習スケジューリングロジックを実装。
- **データベース**: PostgreSQL でユーザー、ルーム、記憶アイテム、復習履歴を永続化。

### 非機能要件

- パフォーマンス: 3D シーンの初回ロード 3 秒以内、API レスポンス 200ms 以内
- セキュリティ: ユーザー認証（JWT）、入力バリデーション、HTTPS 必須
- スケーラビリティ: 同時接続 100 ユーザー

| 競合 | URL | ギャップ |

## マイルストーン

### Phase 0: MVP（2週間）

1. **基盤セットアップ**: CI/CD、リンター、テストフレームワーク
2. **データモデル設計**: ルーム、記憶アイテム、復習履歴のスキーマ
3. **3D ルームエディタ**: Three.js でルーム作成・アイテム配置
4. **間隔反復エンジン**: SM-2 ベースのルールベーススケジューリング
5. **復習 UI + 統計ダッシュボード**: 正答率トラッキング、忘却曲線の可視化

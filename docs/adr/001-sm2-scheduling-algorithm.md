# ADR-001: SM-2 間隔反復アルゴリズムの採用

## ステータス

承認済み (2026-04-01)

## コンテキスト

Memory Palace は記憶アイテムの復習タイミングを自動計算する必要がある。候補となるアルゴリズム:

1. **SM-2 (SuperMemo 2)**: ルールベース。実装が単純で、パラメータが直感的
2. **FSRS (Free Spaced Repetition Scheduler)**: ML ベース。精度が高いがデータ量が必要
3. **Leitner System**: ボックス方式。実装は簡単だが粒度が粗い

## 決定

MVP では **SM-2** を採用する。

## 理由

- ルールベースのため実装・テスト・デバッグが容易
- Anki でも長年使われており、実績が十分
- ユーザーデータが蓄積されていない MVP フェーズでは ML モデルの精度優位性が発揮できない
- SM-2 のパラメータ (ease_factor, interval, repetitions) が学習分析ダッシュボードで直感的に表示できる

## 結果

- `services/scheduling.py` に SM-2 アルゴリズムを実装
- Phase 2 以降、十分なユーザーデータが蓄積されたら FSRS への移行を検討
- 移行時はスケジューリングサービスのインターフェースを維持し、アルゴリズム実装のみ差し替える
